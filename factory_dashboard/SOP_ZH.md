## 1. 目的

本标准操作程序（SOP）规定了**智能工厂看板**系统的每日操作步骤。该看板整合并可视化来自三个核心工厂系统的实时和历史数据：

- **OEE（整体设备效率）** — 各产线的机器生产率数据
- **AGV（自动导引车）** — 物料搬运任务日志
- **AOI（自动光学检测）** — 产品缺陷图像检测结果

目的是确保每位操作员在加载、查看和报告工厂数据时遵循一致、可靠的流程，防止数据错误、系统崩溃或结果误判。

---

## 2. 适用范围

本SOP适用于：所有班次操作员（早/中/夜班）、生产工程师和主管、品质工程师、物流主管。

**不涵盖**：软件安装、后端服务器配置、修改源代码。

---

## 3. 职责分工

| 角色 | 职责 |
|---|---|
| **班次操作员** | 班次开始时加载当日数据；监控KPI；上报异常 |
| **生产工程师** | 验证OEE数据准确性；审查楼层图表；按日期筛选 |
| **品质工程师** | 审查AOI合格率；若 < 90% 则调查FAIL图像 |
| **物流主管** | 监控AGV任务数量；识别最繁忙站点 |
| **IT / 系统管理员** | 维护Python环境；处理软件错误；更新配置 |

## 5. 每日启动程序

> ⚠️ 请按顺序完成每个步骤，不可跳过。

### 第1步 — 打开终端

按 `Win + R` → 输入 `cmd` → 按 **Enter**
```
cd D:\factory_dashboard
```

### 第2步 — 启动看板
```bash
python main.py
```
GUI窗口应在5秒内打开。若未打开，参见[第13节](#13-常见错误与故障排除)。

### 第3步 — 最大化窗口

程序应自动全屏。若未自动全屏，按 `Win + ↑`。

### 第4步 — 验证UI布局

确认以下面板均可见：

| 面板 | 位置 |
|---|---|
| ⚙️ 数据控制 | 左上角 |
| 🔄 AGV总任务数 KPI | 顶部中间 |
| 🖼️ AOI合格率 KPI | 顶部右侧 |
| 最活跃站点（柱状图） | 中左 |
| 动作比例（饼图） | 中间 |
| 每小时流量（折线图） | 中右 |
| OEE F4（柱状图） | 底部左 |
| OEE F5（柱状图） | 底部中 |
| AOI质量详情（饼图） | 底部右 |

## 6. 加载OEE数据

### 第1步 — 点击"📊 Load OEE File"

### 第2步 — 选择文件
- 支持格式：`.xls`、`.xlsx`、`.csv`
- 按住 `Ctrl` 可多选文件

### 第3步 — 等待终端输出

正常完成标志：
```
[OK  ] [OEE] 处理完成，耗时X.XXs – 正在推送数据至UI
```

### 第4步 — 验证图表

- OEE F4 图表显示4楼各产线柱状图
- OEE F5 图表显示5楼各产线柱状图
- 日期筛选下拉框已填充可用日期

### 第5步 — 终端异常检查

| 终端消息 | 处理措施 |
|---|---|
| `[WARN] xlrd failed… trying HTML fallback` | 正常，已自动处理 |
| `[WARN] Column 'OEE' not found` | 确认所选文件正确 |
| `[ERROR] Cannot read 'filename'` | 文件损坏，重新获取 |
| `[ERROR] No valid data` | 联系IT |

## 7. 加载AGV日志数据

### 第1步 — 点击"📜 Load AGV Logs"

### 第2步 — 选择日志文件
- 文件命名格式：`Log2026040108.txt`（`LogYYYYMMDDHH.txt`）
- 全天分析：选择00到23共24个小时文件

### 第3步 — 监控终端输出

```
[OK  ] [AGV] 总任务数：248  |  耗时：0.21秒
[INFO] [AGV] 最活跃站点：[('4C', 80), ...]
[INFO] [AGV] 动作明细：{'DOWN': 124, 'UP': 124}
```

### 第4步 — 验证KPI和图表

| 检查项 | 期望结果 |
|---|---|
| AGV总任务数卡片 | 显示实际任务总数 |
| 最活跃站点图 | 最多10个柱，降序排列 |
| 动作比例图 | UP/DOWN饼图 |
| 每小时流量图 | 折线图，每小时一点 |

### 第5步 — 异常处理

| 情况 | 处理措施 |
|---|---|
| 总任务数 = 0 | 检查文件夹是否正确 |
| 只显示Unknown站点 | 上报IT，更新POINT_MAP |
| 动作比例严重失衡 | 检查日志文件完整性 |

## 8. 加载AOI图像数据

### 第1步 — 点击"🖼️ Select AOI Folder"

### 第2步 — 选择AOI根文件夹
- 系统递归扫描，最深**5层**
- 自动跳过非图像文件

### 第3步 — 监控终端输出

```
[OK  ] [AOI] 总计 → 合格：7 | 不合格：10 | 合格率：41.2%
[INFO] [AOI] 扫描完成，耗时0.01秒 – 正在推送数据至UI
```

### 第4步 — 验证KPI

- 🟢 绿色 = 合格率 ≥ 90%（正常）
- 🔴 红色 = 合格率 < 90%（需立即处理）

### 第5步 — 合格率低时的处理

1. 记录合格/不合格数量
2. 打开AOI文件夹，按文件名排序
3. 含 `"all pass"` = 合格；含 `"fail"` = 不合格
4. 立即通知品质工程师
5. 在日报中记录事件

## 9. 读取与解读看板

### 9.1 KPI卡片基准

| 指标 | 正常值 | 警告阈值 |
|---|---|---|
| AGV总任务数 | 与前日基本一致 | 较前日下降 > 30% |
| AOI合格率 | ≥ 90% | < 90% 立即上报 |

### 9.2 OEE图表解读

- Y轴：OEE百分比（0–100%）
- 世界级水平：≥ 85%
- 警告阈值：< 70%，需调查
- 悬停查看精确值

### 9.3 AGV图表解读

- **站点柱状图**：高任务量站点需确认AGV调度合理
- **动作比例**：DOWN（送料）/ UP（还料），比例应接近50/50
- **每小时流量**：峰值应与班次交接时间吻合，平坦段可能表示停机

## 10. 日期筛选功能使用

- **柱状图**：显示产线名和OEE/任务值
- **饼图**：显示类别、数量和百分比
- **折线图**：显示小时和任务数

## 12. 终端日志监控

终端窗口操作期间必须保持**打开**。

### 日志级别说明

| 颜色 | 级别 | 含义 |
|---|---|---|
| 🔵 青色 | `INFO` | 正常执行步骤 |
| 🟢 绿色 | `OK` | 步骤成功 |
| 🟡 黄色 | `WARN` | 非致命问题，已自动处理 |
| 🔴 红色 | `ERROR` | 失败，需要操作 |

### 需要上报的情况

- 出现 `[ERROR]` 且处理中断
- 图表未更新但终端显示成功
- 应用程序崩溃（显示 `Traceback`）

## 13. 常见错误与故障排除

| 错误 / 现象 | 原因 | 解决方案 |
|---|---|---|
| GUI未打开 | Python环境问题 | `python --version` 检查；联系IT |
| `ModuleNotFoundError` | 依赖未安装 | `pip install PyQt5 pandas matplotlib xlrd openpyxl` |
| 图表空白 | 文件格式或列缺失 | 检查终端 `[WARN]` / `[ERROR]` |
| 只显示F4不显示F5 | 文件中无F5数据 | 确认OEE文件含 樓層='F5' 的行 |
| 弹出"任务进行中"提示 | 按钮点击过快 | 等待当前任务完成 |
| AOI全部为0 | 文件夹选择错误 | 确认文件夹含"all pass"/"fail"命名图像 |
| 应用崩溃 | 运行时错误 | 截图终端，重启程序，联系IT |

使用 `Win + Shift + S` 截图，保存至日报。

### 第2步 — 记录关键指标

| 指标 | 今日数值 |
|---|---|
| AGV总任务数 | ___ |
| 最繁忙站点 | ___ |
| AOI合格率 | __% |
| OEE F4平均值 | __% |
| OEE F5平均值 | __% |
| 异常情况 | 是/否 |

### 第3步 — 关闭应用
`Alt + F4` 关闭窗口，然后关闭终端。

按站点数据保留政策归档处理后的文件。
